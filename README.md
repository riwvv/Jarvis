🎙️ Jarvis — Голосовой ассистент на C# с локальным ИИ и RAG-памятью
Интеллектуальный голосовой помощник для Windows с полной приватностью.
Работает офлайн, понимает русскую речь, управляет приложениями и системой, а главное — обладает долговременной памятью и может использовать её для самообучения.
Всё локально — ничего не уходит в облако.

Jarvis — это полноценный голосовой ассистент, который не требует интернета. Он пробуждается по слову «Джарвис» , распознаёт голосовые команды (Vosk), понимает их с помощью локального ИИ (Ollama + Qwen3/GPT-OSS) и выполняет действия: открывает программы, сайты, управляет громкостью, работает с файлами и отвечает на вопросы. А благодаря встроенной RAG-памяти (Retrieval-Augmented Generation) он запоминает успешные команды и может использовать их в будущем для выполнения похожих задач.

✨ Ключевые возможности
🎯 Голосовое управление
Пробуждение по ключевому слову — просто скажите «Джарвис».

Распознавание речи — офлайн через Vosk (русский язык).

Голосовая активность — ассистент сам понимает, когда вы закончили команду.

🧠 ИИ и автоматизация
Локальный LLM — поддержка Ollama. Используйте модели Qwen3, GPT-OSS, Llama.

Semantic Kernel — умные цепочки действий и контекстные диалоги с помощью плагинов.

Долговременная RAG-память — ассистент автоматически запоминает успешные команды и может искать в памяти информацию о прошлых действиях, чтобы отвечать на вопросы или повторять их.

Плагины как инструменты — все действия (открытие сайтов, управление системой, работа с файлами) и даже сама память вынесены в плагины, которые LLM вызывает по мере необходимости. Это делает архитектуру гибкой и расширяемой.

🚀 Быстрый старт (пошаговая инструкция)
1. Установите и запустите Ollama
Ollama нужен для работы языковой модели (LLM) и создания эмбеддингов для RAG-памяти.

Скачайте Ollama с официального сайта.

Установите его, оставив настройки по умолчанию.

После установки Ollama автоматически запустится в фоне (убедитесь, что он работает — иконка в трее).

Откройте терминал (PowerShell / CMD) и скачайте необходимые модели.

Для работы ИИ: ollama pull qwen3.5:4b (или другую модель на ваш выбор).

Для работы RAG-памяти (создания эмбеддингов): ollama pull qwen3-embedding:4b

Важно: Для стабильной работы RAG и долговременной памяти рекомендуется создать кастомную модель с увеличенным контекстным окном:

bash
ollama run qwen3.5:4b
/set parameter num_ctx 8192
/save qwen3.5:4b-long
В файле конфигурации appsettings.json укажите "ModelId": "qwen3.5:4b-long".

2. Установите модель для распознавания речи (Vosk)
Скачайте русскую модель с официальной страницы Vosk: vosk-model-ru-0.42 (рекомендуемая, около 1.2 ГБ).

Распакуйте архив.

Переименуйте полученную папку в vosk-model-ru-0.42.

Поместите её в папку Models рядом с Jarvis.exe.

text
📁 Jarvis
   📄 Jarvis.exe
   📁 Models
      📁 vosk-model-ru-0.42
3. Настройте приложение
В файле appsettings.json в секции AISettings укажите нужную модель для ИИ и эмбеддингов.

Убедитесь, что параметры Endpoint и ModelId соответствуют вашим настройкам Ollama.

4. Запустите Jarvis
Скачайте последнюю версию Jarvis.exe из раздела Releases (или соберите проект из исходников).

Запустите Jarvis.exe.

При первом запуске может потребоваться разрешение брандмауэра — разрешите. Это нужно для связи с Ollama.

Если появится окно с ошибкой «Ollama не запущена» , вернитесь к шагу 1 и убедитесь, что Ollama работает.

🗣️ Примеры команд
После запуска скажите «Джарвис» и дождитесь сигнала (или просто отдайте команду).

Вы говорите	Джарвис делает
«Открой Chrome»	Запускает браузер.
«Как меня зовут?»	Вспоминает, если вы называли имя ранее (используя RAG).
«Сверни все окна»	Сворачивает все окна на рабочем столе.
«Сделай громче 50 процентов»	Изменяет системную громкость.
«Открой YouTube»	Открывает сайт в браузере по умолчанию.
«Покажи все программы»	Создаёт на рабочем столе файл со списком установленных приложений.
«О чём я тебя сегодня просил?»	Джарвис найдёт в RAG-памяти последние успешные команды и ответит.
«Сделай то же, что и вчера»	Найдёт в памяти последнюю команду и выполнит её.
🧠 Как работает RAG-память
В Jarvis реализован продвинутый, но интуитивно понятный механизм долговременной памяти на основе RAG (Retrieval-Augmented Generation), который работает как инструмент, а не как обязательная часть каждого запроса.

Автосохранение успешных команд: После того как Джарвис успешно выполнит команду (ответ начинается с DONE), она автоматически сохраняется в векторную базу данных вместе с текстом команды.

Поиск по запросу: Когда пользователь задаёт вопрос о прошлом («О чём я тебя просил?»), LLM (мозг ассистента) сама решает, что ей нужна помощь, и вызывает специальный RAG-плагин.

Семантический поиск: RAG-плагин ищет в памяти не по точному совпадению слов, а по смыслу (семантически). Именно поэтому «О чём я тебя просил?» находит диалог про «открой ютуб».

Контекстный ответ: Найденная информация возвращается LLM, которая формулирует на её основе ответ.

Такой подход делает RAG не «обязанностью», а мощным инструментом, который LLM использует осознанно, когда это действительно необходимо.

Вся векторная база данных хранится в файле %LocalAppData%\Jarvis\vectors.json\kb.json.

🧩 Архитектура и плагины
Основная логика работы вынесена в плагины Semantic Kernel. Это позволяет легко расширять функциональность ассистента без изменения его ядра.

Плагин	Назначение	Методы
ApplicationPlugin	Запуск приложений и игр	LaunchApp, LaunchSteamGame, CreateAppListFile
SystemAudioPlugin	Управление громкостью	ChangeVolume, VolumeTurnOff
BrowserPlugin	Открытие сайтов	OpenUrl
FilePlugin	Работа с файлами	CreateFile, ReadFile, DeleteFile и др.
SystemCommandPlugin	Системные команды	MinimizeAllWindows, LockScreen, Shutdown, Restart и др.
PrankPlugin	Шутки	OpenJoke
RagPlugin	Долговременная память	SearchMemory, SaveToMemory
LLM (мозг ассистента) сама решает, какой плагин и с какими параметрами вызвать, анализируя запрос пользователя.

🗂️ Структура проекта
Проект построен на чистой архитектуре с использованием Dependency Injection:

/Services — основные сервисы: распознавание и синтез речи, работа с ИИ и RAG-памятью, иконка в трее и др.

/Plugins — плагины для Semantic Kernel, каждый отвечает за свою зону ответственности.

/Interfaces — контракты для сервисов (например, IRagMemoryService).

/Models — POCO-классы для данных.

/ViewModels и /Views — реализация паттерна MVVM для WPF-интерфейса.

/Extensions — классы для настройки DI-контейнера и расширения функциональности.

/Wrapper — обёртки для взаимодействия с Ollama API.

💬 Устранение неполадок
Ошибка No service for type 'RagMemoryService': Убедитесь, что в AppExtensions.cs метод AddServices() вызывается до AddSemanticKernel(). Порядок регистрации важен.

Ассистент не выполняет команды, а «рассуждает»: Возможно, LLM не может выбрать подходящий плагин. Проверьте системный промпт в CommunicationAiService и убедитесь, что он содержит чёткие инструкции по использованию плагинов.

RAG не находит прошлые команды: Убедитесь, что файл %LocalAppData%\Jarvis\vectors.json\kb.json существует. Проверьте, что модель эмбеддингов (qwen3-embedding:4b) загружена в Ollama.

Ответы модели обрываются: Увеличьте значение num_ctx для модели в Ollama (например, до 8192) и настройте MaxTokens в CommunicationAiService.

Другие ошибки: Логи пишутся в файл logs/jarvis-logs.txt. Изучите их для детальной диагностики.
